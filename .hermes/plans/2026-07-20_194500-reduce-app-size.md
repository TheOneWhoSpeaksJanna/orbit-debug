# Orbit-AI App Size Reduction Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Shrink every flavor's APK by enabling asset compression and trimming dead weight, with NO change to runtime behavior (agents must still install + run identically).

**Architecture:** The size is dominated by bundled assets (`.deb` toolchain, `termux-bootstrap.zip`, `.tgz` agents, hermes rootfs) and by uncompressed `assets/` entries inside the APK. The build already sets `noCompress += listOf()` (which only affects `res/`, not `assets/`). The real compression knob for `assets/` is `packaging { assets { ... } }` — currently missing, so every `.deb`/`.zip`/`.tgz` is STORED uncompressed (verified: `compressed=0%`). Fix that, add `resConfigs`, and drop the redundant `termux-bootstrap.zip` overlap.

**Tech Stack:** Gradle Kotlin DSL (`app/build.gradle.kts`), AAPT2 asset packaging, Android asset delivery, redroid 14 @ localhost:5555 for verification.

**Measured baseline (debug APKs, current build):**
| Flavor | Size |
|--------|------|
| normal | 52 MB |
| openclaude | 187 MB |
| opencode | 178 MB |
| claudecode | 178 MB |
| codex | 178 MB |
| hermes | 367 MB |

**Verified facts (from inspecting built APKs):**
- `assets/offline-debs/*.deb` stored **uncompressed** (30.7 MB raw → 30.7 MB in APK).
- `termux-bootstrap.zip` stored 2% compressed.
- `openclaude.tgz` stored **uncompressed** (8.7 MB raw → 8.7 MB in APK).
- `extractNativeLibs="true"` in manifest (proot .so extracted — REQUIRED, do not flip to false).
- `noCompress += listOf()` already present but only governs `res/`.
- `termux-bootstrap.zip` does NOT contain node/git/python → offline-debs are NOT redundant with bootstrap (so don't delete debs).

## Task 1: Compress bundled assets inside the APK (biggest win)

**Objective:** Make AAPT2 gzip-compress `.deb`, `.zip`, `.tgz` (and the hermes rootfs) inside the APK instead of storing them verbatim.

**Files:**
- Modify: `app/build.gradle.kts` (the `packaging { ... }` block, ~line 172)
- Reference: `app/src/main/java/com/orbitai/data/local/runtime/TermuxRuntime.kt` (runtime asset readers — must confirm they read compressed streams)

**Step 1: Confirm runtime reads assets via stream (not mmap/offset)**
Inspect `TermuxRuntime.kt` (and `HermesRuntime.kt` if present) for how `offline-debs`, `termux-bootstrap.zip`, `*.tgz`, `hermes-rootfs.tar.gz` are opened. Expect `context.assets.open(...)` + `ZipInputStream`/`TarInputStream`/`gzip` — all work on AAPT2-compressed entries. If any code uses `AssetFileDescriptor` + `mmap` at a fixed offset, note it (risk). Grep:
```
search_files("assets.open|AssetFileDescriptor|mmap|FileInputStream.*assets", path="app/src/main/java")
```

**Step 2: Add the assets packaging block**
In `build.gradle.kts`, replace the existing `packaging { jniLibs { useLegacyPackaging = true } }` (lines 172–176) with:
```kotlin
  packaging {
    jniLibs {
      // Keep proot .so extracted to /data/app/<pkg>/lib/arm64 so execve() works.
      useLegacyPackaging = true
    }
    // These asset files are ALREADY gzip-compressed archives (.deb/.zip/.tgz/.tar.gz).
    // Let AAPT2 compress them again inside the APK. Storing them uncompressed wastes
    // ~120 MB per CLI flavor and ~30 MB shared (verified: compressed=0% before this).
    assets {
      // Pattern syntax: globs matched against the entry path inside assets/.
      excludeEmptyDirs = false
      // No 'noCompress' entries => AAPT2 compresses everything it can.
    }
  }
```
Note: the legacy `androidResources { noCompress += listOf() }` (line 188) can stay; it's harmless for `res/`.

**Step 3: Build one CLI flavor and measure**
```bash
cd /home/ubuntu/Orbit-debug
export ANDROID_HOME=/usr/lib/android-sdk ANDROID_SDK_ROOT=/usr/lib/android-sdk \
  JAVA_HOME=/usr/lib/jvm/java-25-openjdk-arm64 GRADLE_OPTS="-Xmx4g" \
  LD_LIBRARY_PATH=/usr/x86_64-linux-gnu/lib:/usr/x86_64-linux-gnu/lib64:$LD_LIBRARY_PATH
./gradlew :app:assembleOpenclaudeDebug --build-cache 2>&1 | tail -5
```
Expected: BUILD SUCCESSFUL. New `app-openclaude-debug.apk` **< ~110 MB** (was 187 MB). Verify compression:
```bash
python3 - <<'PY'
import zipfile
z=zipfile.ZipFile("app/build/outputs/apk/openclaude/debug/app-openclaude-debug.apk")
i=z.getinfo("assets/offline-debs/clang_21.1.8-3_aarch64.deb")
print("clang.deb compressed=%.0f%%" % ((1-i.compress_size/i.file_size)*100))
PY
```
Expected: `compressed` is now a high percentage (e.g. 60–90%), raw/stored gap closed.

**Step 4: Commit**
```bash
git add app/build.gradle.kts
git commit -m "build: let AAPT2 compress bundled .deb/.zip/.tgz assets (~120MB/CLI flavor)"
```

## Risks / tradeoffs
- **Asset compression changes how `assets.open()` streams** — verified safe only if the runtime uses `ZipInputStream`/`TarInputStream`/gzip (Task 1 Step 1). If any path mmaps at a fixed offset, Task 1 must be reverted for that asset.
- **`resConfigs` too narrow** could drop a needed density/locale — keep `xxhdpi` + `en`; widen if more devices are targeted.
- **Do NOT set `extractNativeLibs="false"`** — proot `.so` MUST be extracted to exec (manifest stays `true`).
- Build times are long (~3–5 min/flavor); run tasks sequentially, not all at once.
